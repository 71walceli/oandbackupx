Usage notes

Backup modes: An app can be backed up with only its apk, only its data or both. An app not currently installed can only be restored if the apk is present.

Sicherung löschen: Löscht die Sicherungsdateien für eine bestimmte Sicherungsinstanz. Während "Alle Backups löschen" alle Backups der gewählten App löscht.

Mehrere Benutzer: Multi-User ist noch etwas experimentell, sollte aber funktionieren. Beim Wiederherstellen in einem Mehrbenutzer-Szenario wird `pm install -r $apk` aufgerufen und anschließend wird die Anwendung für jeden Benutzer deaktiviert, der die Anwendung in /data/system/user/$user/package-restrictions.xml aufgelistet hat (es sei denn, die Anwendung ist als enabled="1" aufgeführt). Dies kann für Benutzer, die dieselbe Anwendung zu einem späteren Zeitpunkt installieren, zu Problemen führen, ist aber notwendig, um zu verhindern, dass die Anwendung für alle Benutzer gleichzeitig installiert wird. Das Kontextmenü verfügt über eine Option zum Aktivieren oder Deaktivieren von Anwendungen, die verwendet werden können, wenn andere Benutzer eine bestimmte Anwendung aufgrund der Deaktivierung bei der Wiederherstellung nicht mehr verwenden können. Aktivieren und Deaktivieren funktioniert nur nach einer Erstinstallation (nicht notwendigerweise von OAndBackupx) oder Wiederherstellung der Anwendung.

Aktivieren / Deaktivieren: Verwendet das Android-Skript `pm` zum Aktivieren oder Deaktivieren einer App. Das Deaktivieren einer App entfernt sie aus der normalen Benutzeroberfläche, ohne sie zu deinstallieren. Dies kann verwendet werden, um eine App für mehrere Benutzer gleichzeitig zu aktivieren oder zu deaktivieren (wenn das Gerät mehrere Benutzer aktiviert hat). Benutzer werden mit einer ID identifiziert, z. B. 0 ist der erste Benutzer (Eigentümer), 10 ist normalerweise das Arbeitsprofil.

Mehrere Benutzer: Mehrere-Benutzer ist noch etwas experimentell, sollte aber funktionieren. Das Aktivieren und Deaktivieren funktioniert nur nach einer Erstinstallation (nicht unbedingt von OAndBackupx) oder Wiederherstellung der App.

Systemlose Apps: Das Sichern und Wiederherstellen von systemlosen Apps, die als Magisk-Modul installiert sind, wird nicht unterstützt und das herumprobieren damit könnte zu einem Fehler im System führen und Sie zwingen, neu zu formatieren und zu flashen.

Navigation: Die Navigationsleiste führt zu Einstellungen, Home, Batch-Backup, Batch-Wiederherstellung und Zeitplan-Backups in dieser Reihenfolge. Die Benachrichtigung auf Home bedeutet, dass es eine Reihe von Apps gibt, die seit dem letzten Backup aktualisiert wurden.

Apps mit Keystore: Android verfügt über einen verschlüsselten Speicher für sensible Daten. Apps, die diesen nutzen, um Anmeldedaten oder einige Schlüssel für ihre Funktion zu speichern, können nicht erfolgreich wiederhergestellt werden. Zwei Beispiele für solche Apps sind Signal und Element(Riot).
