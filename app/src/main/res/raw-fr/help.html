Utilisation

Backup modes: An app can be backed up with only its apk, only its data or both. An app not currently installed can only be restored if the apk is present. Three special types of data is supported extra to the main data: device protected data, external data and obb data.

Effacer la sauvegarde : Supprime les fichiers de sauvegarde pour l'app choisie.

Désinstaller : Un peu plus aggressif qu'une désinstallation normale. En plus d'une désinstallation normale via les commandes Android, OAndBackupX supprime les fichiers que l'app aurait pu laisser dans /data/app-lib/.

iActiver / Désactiver : Utilise le script Android `pm` pour activer ou désactiver une app. Désactiver une app la retire de l'interface utilisateur normale sans la désinstaller. Il est aussi possible d'activer ou désactiver une app pour plusieurs utilisateurices à la fois (si l'appareil est configuré pour). Les utilisateurices sont identifié⋅es par un identifiant. Le premier est le numéro 0 (propriétaire).

Utilisateurices multiples : Cette fonctionnalité est quelque peu expérimentale mais devrait marcher. En restaurant dans un scénario multi-utilisateurices, `pm install -r $apk` est appelé puis l'app est désactivée pour chaque utilisateurice ayant cette app listée dans /data/system/user/$user/package-restrictions.xml (sauf si l'app est notée comme enabled="1").
Cela peut mener à des problèmes pour les utilisateurices installant cette même app plus tard, mais est nécessaire pour empécher l'app d'être installée à tous le monde en même temps. Le menu contextuel a une option pour activer ou désactiver les apps, qui peut être utilisé s'il devient impossible pour certain⋅es utilisateurices d'utiliser une app spécifique suite à la désactivation lors de la restauration.
L'activation ou la désactivation fonctionnent uniquement après l'installation initiale (par forcément depuis OAndBackupX) ou après restauration de l'app.

System-less apps: backup and restore of system-less apps installed as Magisk module is not supported and trying it could lead to failure in the system and forcing you to format and flash anew.

Activer / désactiver : utilise le script Android 'pm' pour activer ou désactiver une application. La désactivation d’une application la supprime de l’interface utilisateur normale sans désinstaller. Ceci peut être utilisé pour activer ou désactiver une application pour plusieurs utilisateurs à la fois (si l’appareil a plusieurs utilisateurs activés). Les utilisateurs sont identifiés avec un id par exemple 0 est le premier utilisateur (propriétaire) 10 est généralement le profil de travail.

Apps using Keystore: Android has an encrypted storage for sensitive data. Apps that make use of this to store login data or some keys for its function can't be restored successfully. Two examples of such apps are Signal and Element(Riot).
