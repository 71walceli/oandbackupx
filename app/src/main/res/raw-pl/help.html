Wskazówki dotyczące użytkowania

Backup modes: An app can be backed up with only its apk, only its data or both. An app not currently installed can only be restored if the apk is present.

Delete backup: Deletes the backup files for the chosen app.

Odinstaluj: Nieco bardziej agresywne niż normalne odinstalowanie. Oprócz normalnego odinstalowywania za pomocą poleceń Androida, odinstalowanie z OAndBackupX usuwa pliki, które aplikacja mogła pozostać w /data/app-lib/.

Włącz / wyłącz: Używa Androidowego skryptu 'pm', aby włączyć lub wyłączyć aplikację. Wyłączenie aplikacji powoduje usunięcie jej z normalnego interfejsu użytkownika bez odinstalowywania. Może to służyć do włączania lub wyłączania aplikacji dla wielu użytkowników na raz (jeśli urządzenie ma wielu użytkowników). Użytkownicy są identyfikowane za pomocą identyfikatora. 0 jest pierwszym użytkownikiem (właścicielem).

Wielu użytkowników: Tryb wielu użytkowników jest jeszcze trochę eksperymentalny, ale powinien działać. Podczas przywracania w scenariuszu wieloużytkownikowym, `pm install -r $apk` zostaje wywołany i następnie aplikacja jest wyłączona dla każdego użytkownika, który ma aplikację wymienioną w /data/system/user/$user/package-restrictions.xml (chyba że aplikacja jest wymieniona jako włączona="1").
Może to spowodować problemy dla użytkowników instalujących tę samą aplikację w późniejszym czasie, ale jest konieczne, aby uniemożliwić instalację aplikacji dla wszystkich użytkowników w tym samym czasie. Menu kontekstowe posiada opcję włączania lub wyłączania aplikacji, z której można korzystać, gdy inni użytkownicy nie będą mogli korzystać z konkretnej aplikacji z powodu wyłączenia funkcji przywracania.
Włączanie i wyłączanie działa tylko po pierwszej instalacji (niekoniecznie z OAndBackupx) lub przywróceniu aplikacji.
