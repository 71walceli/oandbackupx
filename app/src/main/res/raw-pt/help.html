Usage notes

Modos de backup: uma app pode ser copiado apenas com o apk dela, apenas com os dados dela ou ambos. Uma app não instalada atualmente só pode ser restaurada se o apk estiver presente.

Delete backup: Deletes the backup files for the chosen app.

Uninstall: Somewhat more aggressive than a normal uninstall. In addition to doing a normal uninstall via android commands, uninstalling from OAndBackupX deletes files the app might have left over in /data/app-lib/.

ativa/desabilita> Usa o script android`pm` para ativar o desabilitar uma app. Desabilitando uma app, remove-a da interface de usuário normal sem desinstalá-la. Isso pode ser usado para ativar ou desabilitar uma app para múltiplos usuário ao mesmo tempo (Se o dispositivo tem múltiplos usuário ativados). Usuários são identificados com uma id 0 é o primeiro usuário (proprietário).

Múltiplos utilizadores: Multi-utilizador ainda é algo experimental mas deve funcionar. Quando restaurar em um cenário multi-utilizadores, `pm install -r $apk` é chamado e, subsequentemente, a app é desativada para todos utilizadores que tem a app listada em /data/system/user/$user/package-restrictions.xml (a não ser que a app esteja listada como enabled="1".
Isso pode criar problemas para utilizadores instalando a mesma app no ponto posterior, mas, é necessário prevenir a app de ser instalada para todos os utilizadores ao mesmo tempo. O menu contexto tem uma opção para ativar ou desativar apps que pode ser usadas se outros utilizadores se tornarem inaptos para usar específica app, devido à desativação na restauração.
Ativar e desativar funciona somente após uma instalação inicial (não necessariamente do OAndBackupx) ou restaurar da app.
