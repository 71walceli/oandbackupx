Примечания по использованию

Backup modes: An app can be backed up with only its apk, only its data or both. An app not currently installed can only be restored if the apk is present.

Delete backup: Deletes the backup files for the chosen app.

Деинсталляция: Нечто более агрессивное, чем обычная деинсталляция. В дополнение обычной деинсталляции через команды android, деинсталляция из OAndBackupX удаляет файлы приложения, которые могли остаться в /data/app-lib/.

Включение / выключение: Использует скрипт `pm` для включения или отключения приложения. Выключение приложения убирает его из интерфейса пользователя без удаления. Может быть использовано для включения или отключения приложения для нескольких пользователей одновременно (если на устройстве включено несколько пользователей). Пользователь идентифицируется через id. 0 - первый пользователь (владелец).

Несколько пользователей: Многопользовательский режим это все еще нечто экспериментальное, но должно работать. При восстановлении в многопользовательском режиме вызывается `pm install -r $apk` и впоследствии приложение отключено для всех пользователей, у которых приложение присутствует в /data/system/user/$user/package-restrictions.xml (если только приложение не отмечено как включенное ="1").
Это может создать проблемы для пользователей, устанавливающих одинаковое приложение позже, но необходимо предотвратить установку приложения для всех пользователей одновременно. В контекстном меню есть опция для включения или отключения приложений, что может быть использовано, если другие пользователи не могут использовать определенные приложения из-за отключения при восстановлении.
Включение и отключение работает только после начальной установки (не обязательно из OAndBackupX) или восстановления приложения.
