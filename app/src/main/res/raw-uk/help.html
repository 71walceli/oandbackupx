Примітки по користуванню

Види запасок: Існують такі різновиди резервних копій (запасок) додатку: тількі файл додатку АРК, тільки дані додатку, або файл разом з даними. Невтановлений додаток може бути відтворений з резервної копії тільки за присутності в ній файла АРК.

Видалити запаску: Видаляє запаску обраного додатку.

Видалення: Дещо більш ретельне видалення, ніж зазвичай. Додатково до звичного процесу видалення додатку системними командами, видалення в OAndBackupX видаляє файли що їх міг залишити додаток в /data/app-lib/.

Задіяти / Приспати: Для зміни стану додатку використовується пропис `pm`. Приспаний додаток не буде видалено з пристрою, проте він зникне з оболонки користувача. Це можна використати для декількох користувачів (якщо на пристрої існують декілька користувачів). Кожен користувач має свій id. 0 це id першого користувача (володар).

Multiple users: Multi-user is still somewhat experimental but should work. When restoring in a multi-user scenario, `pm install -r $apk` gets called and subsequently the app is disabled for every user who has the app listed in /data/system/user/$user/package-restrictions.xml (unless the app is listed as enabled="1").
This can create problems for users installing the same app at some later point, but is necessary to prevent the app from being installed to all users at the same time. The context menu has an option to enable or disable apps which can be used if other users become unable to use a specific app due to disabling on restore.
Enabling and disabling only works after an initial install (not necessarily from OAndBackupx) or restore of the app.
